开始我的博客之旅

2026-05-09 阅读约 4 分钟

第一次听到"费曼学习法"是在大二，教授说：如果你不能用简单的语言把一个概念解释清楚，说明你并没有真正理解它。这句话让我开始思考——我在学习中是否只是在"消费"知识，而非真正内化？

于是有了这个博客。

为什么写博客

知识沉淀。学习新技术时，把过程记录下来，不仅帮助记忆，也是日后查阅的最佳手册。曾经踩过的坑、找到的解决方案，以后再遇到同类问题，自己写的文章往往比搜索结果更贴合自己的情况。

加深理解。把知识转化成文字的过程会暴露很多"假懂"的地方。你以为自己理解了某个概念，直到尝试用文字解释它时才发现漏洞。写作是检验理解深度的最好方式之一。

记录成长。一年后回头看今天写的文章，会清楚地看到自己走了多远。这种可感知的进步是持续学习的动力来源。

分享与贡献。互联网上免费的技术资料让我受益良多，那些写博客、录视频、开源代码的前辈们帮助了无数像我这样的学习者。如果我的记录能帮助到哪怕一个人，就值得。

博客定位与内容规划

这里会覆盖几个主题，基本上是我目前在深入学习的方向：

  • 网络安全：渗透测试笔记、CTF 题解、安全工具使用与配置
  • 前端开发：HTML / CSS / JavaScript 学习记录，项目实战复盘
  • 工具与效率：提升开发体验的工具、终端配置、快捷技巧
  • 随笔思考：关于学习方法、成长心态、技术行业的个人想法

我不会假装自己是专家——这里的很多内容是我边学边写的，难免有错误和不完善的地方。如果你发现问题，欢迎指正。

技术实现

这个博客用最简单的方式构建：纯 HTML + CSS，没有框架，没有构建工具，没有数据库。选择这种方式有几个理由：

  • 加载速度极快，零 JavaScript 依赖
  • 维护简单，直接编辑 HTML 文件即可发布
  • 完全掌控代码，没有黑箱
  • 对于现阶段的文章数量，这已经足够

视觉上采用了毛玻璃（Glassmorphism）风格设计——深色渐变背景 + 半透明模糊卡片。这种设计在视觉上既有层次感，在技术上也很好地展示了 CSS 的 backdrop-filter 特性。

后期如果文章数量增多，可能会迁移到 Hugo 或 Astro 这类静态站点生成器，但现阶段这种方式已经足够。

写给未来的自己

希望一年后再回来读这篇文章时，这个博客已经有了 50 篇以上的内容，技术水平也有了实质性的提升。

路不在别处，就在每天写的那几百个字里。
